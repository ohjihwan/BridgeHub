BridgeHub 프로젝트 구조 및 기능 정리
====================================

2. 서버 역할 및 기능
------------------
2.1 API 서버 (api-server)
- 포트: 7100
- 주요 기능:
  * 이메일 인증
    - 인증 코드 생성 및 이메일 전송
    - 인증 코드 검증
    - JWT 토큰 발급
  * 스터디 관리
    - 스터디 생성/조회/수정/삭제
    - 스터디 참여/탈퇴
    - 스터디 메시지 저장
  * 파일 관리
    - 파일 업로드
    - 파일 다운로드
  * 보안
    - JWT 토큰 검증
    - CORS 설정

2.2 소켓 서버 (socket-server)
- 포트: 7500
- 주요 기능:
  * 실시간 통신
    - 채팅 메시지 전송/수신
    - 채팅방 참여/퇴장 관리
    - 참여자 수 실시간 업데이트
  * 시스템 메시지
    - 사용자 입장/퇴장 알림
    - 에러 메시지 전송

2.3 프론트엔드 서버 (front-server)
- 주요 기능:
  * 회원가입
    - 이메일 입력
    - 인증 코드 확인
    - 회원 정보 입력
  * 스터디 관리
    - 스터디 목록 조회
    - 스터디 생성/참여
    - 실시간 채팅
  * 파일 관리
    - 파일 업로드
    - 파일 다운로드

3. API 엔드포인트
---------------
3.1 이메일 인증 (/api/email)
- POST /send-verification: 인증 코드 전송
- POST /verify-code: 인증 코드 확인

3.2 스터디 관리 (/api/study)
- POST /create: 스터디 생성
- GET /list: 스터디 목록 조회
- GET /:id: 스터디 상세 정보
- POST /:id/join: 스터디 참여
- POST /:id/leave: 스터디 탈퇴
- POST /:id/messages: 메시지 저장
- GET /:id/messages: 메시지 조회

4. 소켓 이벤트
------------
4.1 클라이언트 -> 서버
- join-chat: 채팅방 참여
- send-message: 메시지 전송

4.2 서버 -> 클라이언트
- new-message: 새 메시지 수신
- member-count: 참여자 수 업데이트
- error: 에러 메시지

5. 보안 기능
----------
- JWT 기반 인증
- CORS 설정
- 파일 업로드 검증
- 이메일 인증 코드 만료 (10분)

6. 데이터베이스
------------
- 현재: 메모리 기반 임시 저장
- 향후: MongoDB 연동 예정

- 스터디 검색 기능

- 파일 공유 기능 개선
- UI/UX 개선
